## 1. URL 라우팅 구조

### 인증 전
```
/login                          로그인
/register                       회원가입 (워크스페이스 생성)
/forgot-password                비밀번호 재설정
/invite/:token                  고객 초대 수락 (간편 가입)
```

### 관리자 / 멤버 (실제 구현 URL, 2026-04-15 갱신)

> 본 문서의 `/app/*` prefix 는 과거 설계 제안이었고, **현재 실제 구현은 prefix 없음**. 아래는 실제 라우트.

```
/                               → /dashboard 리다이렉트
/dashboard                      대시보드 (placeholder)
/talk                           Q Talk — 워크스페이스의 전체 프로젝트 첫 진입 (기본 3단 레이아웃)
/talk/p/:projectId              Q Talk — 프로젝트 진입 (기본 채널 선택)
/talk/p/:projectId/c/:conversationId  Q Talk — 특정 대화 채널
/talk/p/:projectId/info         Q Talk — 프로젝트 상세/설정
/talk/p/:projectId/candidates   Q Talk — 업무 후보 히스토리 (AI 재호출 없이 저장분 조회)
/talk/p/:projectId/kb           Q Talk — 대화 자료 (프로젝트 스코프)
/talk/p/:projectId/kb/faq       대화 자료 — Pinned FAQ
/tasks                          Q Task — 할일 목록
/tasks/:taskId                  Q Task — 할일 상세
/notes                          Q Note — 회의 목록
/notes/:sessionId               Q Note — 회의 상세
/files                          Q File — 자료함
/bills                          Q Bill — 청구서 목록
/profile                        내 프로필
/settings                       워크스페이스 설정 (기본 Brand 탭)
/settings/brand                 브랜드 탭
/settings/legal                 법인 탭
/settings/language              언어/타임존
/settings/members               멤버/역할 매핑 (Cue 포함)
/settings/cue                   Cue 모드·사용량
```

**네비게이션 규칙**
- 로그인 후 기본 진입: `/talk` (Phase 5 완료 후) 또는 `/dashboard` (placeholder 상태)
- 워크스페이스 선택: 로그인 직후 속한 워크스페이스 ≥ 2개면 `/workspaces` 선택 화면, 1개면 자동 진입
- 워크스페이스 스위처: 우측 상단 드롭다운 (전역 접근)
- Ctrl+K: 좌측 검색 바 포커스 (모달 아님)

### Client (고객)
```
/                               → /talk 자동 리디렉션
/talk                           내가 참여한 프로젝트 리스트 (customer 채널만)
/talk/p/:projectId/c/:conversationId  고객 채널 진입
/talk/p/:projectId/info         프로젝트 상세 (읽기 전용 요약)
/tasks                          내 업무 (본인 담당 필터)
/profile                        내 프로필
/settings                       언어/알림 토글
```

**고객 전용 가시성 규칙**
- 사이드바: 언어 / 프로필 / 로그아웃 3개만 (Q Talk 주 사용처)
- 프로젝트 리스트: 1개면 기본 접힘, 2개 이상이면 펼침
- 채널: customer 만, internal 숨김
- 내부 메모 / Cue 답변 대기 카드 / 업무 추출 토글 / 업무 후보 섹션 — 전부 숨김
- 개인 메모: 가능 (본인만 보임)

### 2.2 Q Talk — 프로젝트 중심 3단 레이아웃 (2026-04-15 전면 재작성)

**구조**: 좌측 프로젝트 리스트 / 가운데 채팅 / 우측 탭 (5 섹션 아코디언).
좌/우 패널은 Q Note 스타일로 **접기/펼치기 + 리사이즈** 지원.

```
┌──────────┬────────────────────────────┬────────────────┐
│ 좌측     │ 가운데 (채팅)               │ 우측 탭         │
│          │                             │                │
│ [워크스페이스] [⚙]│ [프로젝트명 > 채널] [◀ 우측탭 토글]│ [주요 이슈]   │
│ [검색 Ctrl+K] │                             │ [내 할 일]    │
│          │ ┌ 내부논의 │ 고객소통 │ + ┐  │ [프로젝트 업무]│
│ 프로젝트 ▼│ ─────────────────────────── │ [프로젝트 메모]│
│ ▪ 프로젝트A│                             │ [프로젝트 정보]│
│   ↳ 고객소통 ●│ [아바타] 김영희 14:31      │                │
│ ▪ 프로젝트B│  로고 시안 3종 정리 중       │ (리사이즈)     │
│ ▪ 프로젝트C│                             │                │
│          │ [Cue 뱃지] Cue 14:32         │                │
│ ─────────│  FAQ 에서 찾았습니다         │                │
│ 필터:    │  출처: FAQ #12               │                │
│ 전체|미읽음│                             │                │
│ |내 할 일 │ ─ 업무 후보 5개 생성됨 ─      │                │
│          │                             │                │
│          │ ┌── 질문 카드 ────────┐     │                │
│          │ │ [고객] 배송비 얼마? │     │                │
│          │ │ ┌ Cue 답변 대기 ──┐│     │                │
│          │ │ │ "3천원입니다"   │     │                │
│          │ │ │ [처리][수정][X]│     │                │
│          │ │ └──────────────┘│     │                │
│          │ └────────────────────┘     │                │
│          │                             │                │
│ ─ 접기 ▶ │ ─ 자동추출 [ON] [업무 추출] │ ◀ 접기 ──     │
│          │ [입력창                   ] │                │
│          │ Cue 대기 3개 ↑ [전송]       │                │
└──────────┴────────────────────────────┴────────────────┘
```

**좌측 리스트**
- 상단: 워크스페이스 스위처 / 검색 바 (Ctrl+K 포커스)
- 중앙: 프로젝트 리스트, 각 항목에 읽지 않은 배지·최근 메시지·Cue 활동 점
- 필터: 전체 / 읽지 않음 / 내 할 일
- 검색 활성화 시 리스트 영역이 **인라인 결과 뷰로 전환** (모달 없음)
- Q Note 스타일 접기/펼치기 (localStorage 저장)
- 고객 뷰: 기본 접힌 상태, 프로젝트 ≥ 2 일 때 펼침

**가운데 (채팅)**
- 상단: 프로젝트명 > 채널 탭 (내부논의 / 고객소통 / + 추가)
- 흐름:
  - 일반 메시지: 아바타 + 이름 + 시각 + 본문
  - Cue 메시지: Cue 뱃지 + 출처 인용 (접힘/펼침)
  - 질문 메시지: 코랄 보더 카드 + (담당자 뷰에만) 하단 "Cue 답변 대기" 소형 카드
  - 시스템 메시지: 업무 추출 알림, 고객 초대, 상태 변경
- 하단:
  - 입력창 (메시지 + 파일 첨부)
  - Cue 답변 대기 뱃지 "N 개 ↑" (담당자만)
  - 자동 추출 on/off 토글 (채널별)
  - [업무 추출] 수동 버튼

**우측 탭 (5 섹션 아코디언)** — 각 섹션 헤더 클릭으로 접기/펼치기

| 섹션 | 기본 | 내용 | max-height |
|---|---|---|---|
| 주요 이슈 | 펼침 | 최신 3 개 + [+ 추가] + 인라인 편집 (수동 CRUD, AI 없음) | 180px |
| 내 할 일 | 펼침 | 본인 담당 업무 체크박스, 상태 뱃지, 반복 아이콘, [Q Task 전체] 링크 | 280px |
| 프로젝트 업무 | 접힘 | 전체 업무 최신순, 담당자 표시 없음, 반복 아이콘 | 320px |
| 프로젝트 메모 | 펼침 | 최신 3 + 입력창 + 개인/내부 토글 + 과거 스크롤 | 260px |
| 프로젝트 정보 | 접힘 | 이름/고객사/기간/상태/기본 담당자/역할 매핑 요약 | auto |

**우측 탭 조작**
- 리사이즈 핸들 (가로 드래그)
- 접기 버튼
- **2열 확장 없음** — 섹션 접기 + 내부 스크롤로 높이 부담 해결
- 모바일: 하단 시트 (섹션 구조 동일)

**표시 원칙**
- 발신자 뱃지: 사람 이름 or Cue (이모지 금지 — 텍스트/SVG 아이콘만)
- Cue 답변 출처 인라인 표시 (접힘/펼침)
- Draft 답변은 채팅창에 표시 안 함. 질문 아래 "Cue 답변 대기 카드" 에만 (담당자 뷰)
- 질문 답변 처리: 원질문의 `reply_to_message_id` 로 자동 연결
- 낙관적 잠금으로 동시 처리 충돌 방지

- `/talk/p/:newId` 로 자동 이동 — customer 채널 기본 선택

### 2.7 Client (고객) 화면 — 같은 뼈대, 권한 필터

**원칙**: 고객도 멤버와 **완전히 같은 3단 Q Talk 레이아웃**을 쓴다. 별도 화면 만들지 않음. 권한 필터로 차이 생성.

```
┌──────────┬────────────────────────────┬────────────────┐
```
