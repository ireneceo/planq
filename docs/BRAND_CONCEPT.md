# PlanQ 브랜드 컨셉 (최종)

> **슬로건:** 일을 일답게 하다, PlanQ / 일이 일이되지 않게, PlanQ
> **한 줄:** 업무 전용 고객 채팅 + 실행 구조 통합 OS

---

## 1. 브랜드 네이밍

### PlanQ
**Plan + Q (Cue + Queue)**

계획(Plan)을 실행에 넣는(Q) 플랫폼.

### Q의 이중 의미

| 의미 | 설명 | 브랜드 적용 |
|------|------|------------|
| **Cue (메인)** | 당구 큐 — 공(업무)을 정확히 골(실행)에 넣는 도구. 영화 "큐!" — 실행 시작 신호. | 업무를 정확히 실행으로 전환하는 도구 |
| **Queue (서브)** | 대기열 — 체계적으로 줄 세우는 시스템 | 업무 요청이 체계적으로 정리되는 시스템 |

---

## 2. 슬로건

### 메인 슬로건 (브랜드 정체성)
> **일을 일답게 하다, PlanQ**

일이 체계적으로, 제대로 돌아가게.

### 서브 슬로건 (마케팅/공감)
> **일이 일이되지 않게, PlanQ**

일(work)이 일(burden/고생)이 되지 않게. 고객의 고통에 공감.

### 사용 예시

**랜딩 페이지**
```
PlanQ

일을 일답게 하다.

일이 일이되지 않게 —
요청, 대화, 할일, 자료, 청구까지 한 곳에서.
```

**로그인 화면**: "일을 일답게 하다, PlanQ"
**광고/마케팅**: "일이 일이되지 않게, PlanQ"

## 3. Q 시리즈 (기능 네이밍)

| 기능명 | 역할 | 의미 |
|--------|------|------|
| **Q Talk** | 고객 대화 (채팅) | 대화의 큐 — 대화가 곧 실행 신호 |
| **Q Task** | 할일 관리 | 실행의 큐 — 할일을 정확히 골인 |
| **Q Note** | 실시간 회의 보조 + 음성/회의 정리 | 정리의 큐 — 회의를 핵심만 정확히 |
| **Q File** | 공유자료함 | 자료의 큐 — 필요한 자료를 한 곳에 |
| **Q Bill** | 청구서 | 정산의 큐 — 청구를 깔끔하게 처리 |

확장 가능: Q Report, Q Analytics, Q Calendar 등 자연스럽게 추가.

## 4. 브랜드 포지셔닝

### 한 줄 정의
**업무 전용 고객 채팅 + 실행 구조 통합 OS**

### 우리는 채팅툴이 아니다

| 시장의 인식 | PlanQ의 정체성 |
|------------|---------------|
| "또 다른 메신저" | 업무 실행 OS |
| "고객 응대 도구" | 프로젝트 실행 구조화 시스템 |
| "채팅 + 할일 정리" | 분쟁 방지 + 계약 실행 + 리스크 관리 |

### 핵심 메시지 (3줄)
1. **대화가 곧 계약이다** — 모든 합의가 자동 기록
2. **요청이 곧 업무다** — 메시지 한 번에 할일 전환
3. **기록이 곧 증빙이다** — 분쟁 시 누가 언제 무엇을 했는지 추적

### 경쟁 서비스와의 차별점

| 기존 (채널톡, Zendesk 등) | PlanQ |
|--------------------------|-------|
| 고객 문의 응대 | 업무 요청 → 실행 |
| 대화 기록만 남음 | 대화 = 계약 범위 = 증빙 |
| 할일은 별도 툴 | 메시지에서 바로 할일 전환 |
| 승인/보고 따로 | 한 공간에서 전부 처리 |
| AI 없음 | Q Note로 실시간 회의 보조 + 자동 요약/답변 |

## 5. 타겟 페르소나

### Primary: 전문서비스 대표 (직원 5~30명)
- "카톡으로 업무 지시 받는 게 지겨워요"
- "고객이 '그때 그거 해달라고 했잖아요' 하면 증거가 없어요"
- "프로젝트 10개 동시에 돌리면 뭐가 뭔지 모르겠어요"

**대상 업종:** 광고대행사 / 개발회사·IT 외주 / 브랜딩 스튜디오 / 건설·인테리어 / 컨설팅·회계·법무

**공통점:** "증빙"과 "합의 기록"이 사업 리스크를 결정한다.

### Secondary: 사업자 소속 직원
- "오늘 할 일이 카톡 3개, 메일 2개, 구두 1개에 흩어져 있어요"
- "담당자가 누군지 모르겠어요"

### Tertiary: 고객 (사업자의 클라이언트)
- "내 요청이 처리되고 있는지 모르겠어요"
- "자료를 어디에 보냈는지 찾을 수가 없어요"

## 6. 톤 & 보이스

| 항목 | 방향 |
|------|------|
| 톤 | 전문적이면서 친근 (딱딱하지 않은 프로) |

| 빈 상태 | 행동 유도 ("아직 대화가 없습니다. 고객을 초대해보세요.") |

## 10. 사업 전략 요약

### A전략: 내부 사용 → SaaS 확장
```
Step 1: 우리 회사 내부 시스템으로 먼저 사용
Step 2: 실제 고객과 운영하며 검증
Step 3: 성공 사례 확보 후 SaaS 오픈
```

### 수익 모델
- 월 구독료 (목표: 월 10~15만원)
- 한국 법인 대상: 청구서 발송 → 입금 확인 → 세금계산서 별도
- 채팅툴 가격이 아닌 "실행 통제 시스템" 가격

### 고객 진입 조건
1. 로그인 복잡하면 안 됨
2. 앱 설치 요구하면 안 됨
3. 사용법 설명 필요하면 안 됨
→ **웹 링크 클릭 즉시 채팅 가능, 카톡만큼 쉬워야 함**
