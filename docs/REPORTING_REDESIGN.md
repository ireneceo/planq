# PlanQ 업무 보고서 — UI/콘텐츠 재설계 (확정용)

> 목적: **보고서는 "실제 업무 내용을 파악"하는 문서.** 통계(KPI·health·차트)는 Insights에서 본다.
> 보고서 본문 = 무슨 업무를 했는지, 이슈·담당자·진행 메모·PM 서술·산출물. 숫자는 헤더 한 줄만.

---

## 1. 정보구조 (IA) · 메뉴 구성

보고서는 **세 곳**에 산다 — 편집은 출처에서, 통합은 모아서 본다.

| 위치 | 메뉴 이름 | 권한 | 역할 | 편집/보기 |
|------|----------|------|------|----------|
| Q Task | **나의 보고서** | 본인 | 내가 한 업무 보고 | 작성·확정 (편집) |
| 프로젝트 상세 | **보고서** (프로젝트 내 탭) | PM·멤버 | 그 프로젝트 업무 보고 | 작성·확정 (편집) |
| Q Task | **통합 보고서** | owner·admin | 전사 — 확정본 롤업 | 보기 전용 + 통합확정 |

각 보고서 안에서 상위 구분은 **기간 = [주간 보고서] · [월간 보고서]**.

```
Q Task 좌측 탭
 ├ … (업무 탭들)
 ├ 나의 보고서        [주간][월간]   ← 본인 작성
 └ 통합 보고서        [주간][월간]   ← owner/admin, 보기 + 통합확정
프로젝트 상세 페이지 탭
 ├ 캔버스 · 업무 · 거래 …
 └ 보고서             [주간][월간]   ← PM 작성  (NEW)
```

---

## 2. 보고서 콘텐츠 구성 표준 (공통 골격)

모든 보고서(나의/프로젝트/통합)는 같은 골격. **본문은 실제 업무 내용.**

```
┌ 헤더 ────────────────────────────────────────────────┐
│ 대상명 · 기간 · 상태(초안/확정·자동/수동)                  │
│ 한 줄 지표:  진행 54% · 완료 8 · 진행 6 · 이슈 2 · 지연 3  │ ← 숫자는 여기 한 줄만
├ 요약 (서술) ──────────────────────────────────────────┤
│ 책임자 코멘트(PM/본인) — 이번 기간 핵심·판단·주요 메모        │ ← AutoSave 편집(출처에서만)
├ 본문 (실제 업무 내용) ────────────────────────────────┤
│ ✓ 완료한 업무     업무 제목 · 담당자 · (결과/설명)          │
│ ▷ 진행 중         업무 제목 · 담당자 · 진행률              │
│ ⚠ 이슈 · 리스크   이슈 내용(텍스트) · 지연 업무 제목·담당    │
│ 📄 산출물          발행 문서/파일 제목 · 링크               │
│ → 다음 계획       차기 계획 업무 제목 · 담당               │
│ 👥 이해관계자      담당자 · 참여 고객 (프로젝트)            │
└──────────────────────────────────────────────────────┘
```

**프로젝트 보고서는 프로젝트 대시보드(캔버스) 내용을 그대로 재사용**:
전략 핵심메시지 · 워크스트림(추진과제) 진행 · 이슈 · 이해관계자 · 산출물 · 업무.

## 3. 화면 ① 통합 보고서 (Q Task · 보기 전용)

```
╔══════════════════════════════════════════════════════════════╗
║ [주간 보고서]  [월간 보고서]                                     ║  상위 탭 = 기간
╟──────────────────────────────────────────────────────────────╢
║ ‹  2026년 6월 3주차  ▾  ›                  통합: 초안   [통합 확정] ║  기간 이동(1~2년: 화살표+점프) · owner만 확정
║ 워프로랩 통합 보고서 · 6/15~6/21                                  ║
║ 한 줄:  완료 13 · 진행 24 · 이슈 5 · 산출물 3 · 지연 9            ║
╟──────────────────────────────────────────────────────────────╢
║ 보기:  [● 프로젝트별]   [ 개인별 ]   [ 부서별 ]                    ║  통합보고서 안의 뷰 토글
╠══════════════════════════════════════════════════════════════╣
║ 〈프로젝트별〉 — 프로젝트들이 쫙 나열, 각 프로젝트가 대시보드 내용     ║
║                                                                ║
║  ▌클라이언트 온보딩 자동화          진행 54% · ●위험 · PM 김미정   확정 ║
║    핵심메시지  "온보딩 3일 → 당일"                                ║
║    PM 보고     SSO 연동 일정이 밀려 베타 일정 1주 조정. 핵심 흐름은…  ║
║    ✓ 완료(4)   계정 자동생성 API(이디자) · 안내메일 템플릿(박개발)…  ║
║    ▷ 진행(3)   권한 매핑 검수(김오너 70%) · …                     ║
║    ⚠ 이슈      "외부 SSO 일정 지연"  · 지연 업무 2건               ║
║    📄 산출물    온보딩 가이드 v2 · API 명세서                      ║
║    → 다음      베타 고객 5곳 적용                                 ║
║  ───────────────────────────────────────────────────────────  ║
║  ▌International Onboarding 2026 Q2  진행 34% · ●위험 · PM 김미정 초안║
║    … (같은 구성, 접기/펼치기)                                     ║
║                                                                ║
║  · 미확정 프로젝트는 "초안(자동)"으로 표시, 빈 화면 없음             ║
╚══════════════════════════════════════════════════════════════╝
```

- **〈개인별〉**: 멤버들이 쫙 나열 — 각자 `완료한 일 / 진행 중 / 이슈·지연 / 다음`.
- **〈부서별〉**: 부서들이 쫙 나열 — `부서 요약 → 멤버별 핵심 업무 묶음 → 부서 이슈·산출물`.
- 프로젝트/멤버/부서 항목 클릭 → 그 단위 **전체 보고서**(편집은 출처에서)로 이동.
- 여기서 owner/admin 은 **[통합 확정]** 1회만 (설정 ON 시). 개별 편집 없음.

## 4. 화면 ② 프로젝트 보고서 (프로젝트 상세 · "보고서" 탭, PM 편집)

```
[주간 보고서] [월간 보고서]
‹ 6월 3주차 ›                         상태: 초안   [확정]
─────────────────────────────────────────────
클라이언트 온보딩 자동화 · 6/15~6/21 · 진행 54% ●위험 · D-12
─────────────────────────────────────────────
PM 업무보고  [✎ 자동저장]
  (책임자가 이번 기간 진행·판단·주요 메모를 서술)
─────────────────────────────────────────────
✓ 완료한 업무 (4)     · 계정 자동생성 API — 이디자
▷ 진행 중 (3)         · 권한 매핑 검수 — 김오너 (70%)
⚠ 이슈 · 리스크        · 외부 SSO 일정 지연  · 지연 2건
📄 산출물              · 온보딩 가이드 v2 (문서)
→ 다음 계획            · 베타 고객 5곳 적용
👥 이해관계자          담당 김미정 · 참여 고객 ABC사
```
- 자동 초안(업무 데이터·캔버스에서 자동 집계) → PM 이 코멘트·보정 → **[확정]**.
- 확정본이 통합 보고서로 롤업.
- **프로젝트마다 주간·월간 보고서 리스트**(과거 확정본 열람).

## 5. 화면 ③ 나의 보고서 (Q Task · 본인 편집)

```
[주간 보고서] [월간 보고서]
‹ 6월 3주차 ›                         상태: 초안   [확정]
─────────────────────────────────────────────
내 코멘트  [✎ 자동저장]
─────────────────────────────────────────────
✓ 내가 완료한 일      · 안내메일 템플릿 — (프로젝트: 온보딩)
▷ 진행 중            · 권한 매핑 검수 (70%)
⚠ 내 이슈 · 지연      · 지연 1건
→ 다음 계획          · …
```
- 본인 업무에서 자동 초안 → 코멘트 → 확정. 확정본이 통합 보고서 〈개인별〉로 롤업.

## 6. 통계 ↔ 보고서 경계 (명확히)

| | 보고서 (여기) | 통계 (Insights) |
|---|---|---|
| 목적 | **무슨 업무를 했나** (내용) | 추세·분포·예측 (수치) |
| 형태 | 업무 제목·이슈·서술·산출물 | KPI 카드·차트·health 막대·매트릭스 |
| 숫자 | 헤더 한 줄만 | 메인 |

→ 보고서 화면에서 **KPI 카드·health 막대·매트릭스 테이블 제거.**

## 7. 데이터 출처 (자동 초안)

| 콘텐츠 | 출처 |
|--------|------|
| 완료/진행/지연/차주 업무 | tasks (상태·completed_at·planned_week_start·assignee) |
| 이슈·리스크 | project_issues + 지연 업무 |
| 산출물 | posts(발행) + documents |
| 전략 핵심메시지·워크스트림 | projects 전략 컬럼 + project_workstreams (캔버스) |
| PM/본인 서술 | report_units.narrative |
| 담당자·이해관계자 | project_members + project_clients |

확정 시점 스냅샷 박제 → 이후 수치 변동 무관.

---

## 8. 구현 순서 (확정 후)

1. 백엔드: 프로젝트 보고 자동초안에 **전략·워크스트림·이슈·산출물·이해관계자·진행중** 복원(캔버스 재사용). 멤버/부서 보고에 실제 업무 목록.
2. 프론트 ②③ 편집 화면(나의 보고서 / 프로젝트 보고서 탭) — 내용 중심.
3. 프론트 ① 통합 보고서 — 주간/월간 탭 + 프로젝트별/개인별/부서별 뷰(보기 전용) + 통합확정.
4. 통계 잔재 제거 · 탭 네이밍 정리.
