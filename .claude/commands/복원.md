# 시스템 복원 명령어

---

## Claude 필수 수행 사항

**이 명령어가 실행되면 Claude는 반드시 Irene에게 아래 질문을 한다:**

### 1단계: 서버 선택
```
어느 서버를 복원하시겠습니까?
1. 개발서버 (87.106.11.184) — 코드 + DB
2. 개발서버 DB만
3. 개발서버 코드만 (git reset)
```

### 2단계: 백업 목록 표시
```bash
# 개발서버 일일 백업 목록
ls -lh /opt/planq/backups/dev-daily/ 2>/dev/null | tail -7

# 개발서버 DB 백업 목록
ls -lh /var/backups/planq-db/daily/ 2>/dev/null | tail -7

# Git 최근 커밋 (코드 복원용)
cd /opt/planq && git log --oneline -10
```

### 3단계: 복원 실행 (Irene 승인 후)

#### 개발서버 DB 복원
```bash
# 1. 현재 DB를 먼저 백업 (복원 전 안전장치)
cd /opt/planq/dev-backend
source <(grep -E "^DB_" .env | sed 's/^/export /')
mysqldump -u $DB_USER -p$DB_PASSWORD --single-transaction --quick $DB_NAME | gzip > /var/backups/planq-db/daily/planq_pre_restore_$(date +%Y-%m-%d_%H%M%S).sql.gz

# 2. 선택한 백업으로 복원
gunzip -c [선택한_백업_파일.sql.gz] | mysql -u $DB_USER -p$DB_PASSWORD $DB_NAME

# 3. 서버 재시작
pm2 restart planq-dev-backend
```

#### 개발서버 코드 복원
```bash
cd /opt/planq
# 현재 변경사항 백업
git stash
# 선택한 커밋으로 복원
git reset --hard [커밋해시]
# 프론트 재빌드
cd dev-frontend && npm run build
# 빌드 결과 복사
sudo rm -rf /opt/planq/dev-frontend-build/static 2>/dev/null
sudo cp -r dist/* /opt/planq/dev-frontend-build/
# 백엔드 재시작
pm2 restart planq-dev-backend
```

### 4단계: 복원 확인
```bash
# API health check
curl -s http://localhost:3003/api/health

# DB 테이블 수 확인
cd /opt/planq/dev-backend
source <(grep -E "^DB_" .env | sed 's/^/export /')
mysql -u $DB_USER -p$DB_PASSWORD $DB_NAME -e "SELECT COUNT(*) as table_count FROM information_schema.tables WHERE table_schema = '$DB_NAME';"
```

---

## 절대 금지
- **복원 전 현재 상태를 반드시 백업**
- **복원 후 반드시 health check 실행**
- **POS 관련 DB/파일 절대 건드리지 않기**
