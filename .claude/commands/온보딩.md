# 온보딩 — 새 실행자(개발자/AI) 첫 진입 명령어

이전 세션·이전 실행자와 무관하게, **0에서 안전하게 기여 가능한 상태**로 만드는 스킬.
`/개발시작` 이 "오늘 작업 재개"용이라면, `/온보딩` 은 "처음 온 사람/AI"용이다.

---

## Claude 필수 수행 (순서대로, 생략 금지)

### 1단계: 단일 진입 문서 읽기 (Read 도구)

```
- /opt/planq/docs/ONBOARDING.md   (시스템 지도 + SSOT 지도 + 가드 3축 + 생명선 지도 + 함정 목록)
- /opt/planq/CLAUDE.md            (불변식 원본 — 전체)
- /opt/planq/DEVELOPMENT_PLAN.md  (앞 100줄 — 최근 사이클 흐름)
```

### 2단계: 가드 3축 green 확인 (Bash 실행 — 실측 없이 "정상"이라 말하기 금지)

```bash
cd /opt/planq/dev-backend && node /opt/planq/scripts/health-check.js --quiet
node /opt/planq/scripts/guard-invariants.js
node /opt/planq/scripts/e2e/run.js --suite l1,tenant
```

- 셋 다 exit 0 → "기준선 green" 보고 후 진행
- 하나라도 실패 → **내가 오기 전부터 깨진 것**. 작업 시작 전에 사용자에게 보고 (내 변경으로 오인 방지 + 깨진 채 위에 쌓기 금지)

### 3단계: 작업 맥락 파악

```
- /opt/planq/.claude/session-state.md 읽기 (진행 중 작업 유무)
- git -C /opt/planq log --oneline -10 (최근 커밋 흐름)
```

### 4단계: 온보딩 요약 보고

사용자에게 보고:
1. 시스템 한 줄 요약 (3 프로세스: backend 3003 / qnote 8000 / frontend 빌드)
2. 가드 3축 결과 (✓/✗)
3. 진행 중이던 작업 + 다음 예정 작업 (DEVELOPMENT_PLAN 기반, 구체적으로)
4. 이번 작업이 생명선 영역(ONBOARDING.md §4)에 걸리는지 여부

---

## 절대 금지

- 문서 안 읽고 코딩 시작
- 가드 실행 없이 "정상 상태입니다" 보고
- 기준선이 깨져 있는데 그 위에 새 작업 쌓기
- "어떤 작업을 도와드릴까요?" 식 되묻기 — 다음 작업을 스스로 제안
