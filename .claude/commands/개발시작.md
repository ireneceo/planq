## Claude 필수 수행 사항 (맨 먼저 실행)

**이 명령어가 실행되면 Claude는 반드시 다음을 순서대로 수행해야 합니다:**

### 0단계: 이전 세션 상태 확인
```
Read 도구로 다음 파일 읽기:
- /opt/planq/.claude/session-state.md
```
- 이전에 진행 중이던 작업이 있는지 확인
- 있으면 사용자에게 "이전에 [작업명] 진행 중이었습니다. 이어서 할까요?" 질문

### 1단계: 프로젝트 구조 파악 (Bash로 실행)
```bash
# 핵심 모델 목록 (DB 스키마 이해)
ls /opt/planq/dev-backend/models/

# 핵심 라우트 목록 (API 패턴 이해)
ls /opt/planq/dev-backend/routes/

# 프론트 페이지/컴포넌트 구조
ls /opt/planq/dev-frontend/src/components/ 2>/dev/null
ls /opt/planq/dev-frontend/src/contexts/ 2>/dev/null
```

### 2단계: 핵심 문서 읽기 (반드시 Read 도구로 실행)
```
Read 도구로 다음 파일들 읽기:
- /opt/planq/CLAUDE.md (프로젝트 규칙 — 빌드 명령어, 보안, 검증 절차 등 모든 규칙의 원본)
- /opt/planq/DEVELOPMENT_PLAN.md (개발 로드맵 + 작업 히스토리, 앞부분 200줄)
- /opt/planq/dev-frontend/UI_DESIGN_GUIDE.md (UI 디자인 가이드, 있는 경우)
```
⚠️ CLAUDE.md를 읽지 않고 코딩을 시작하는 것은 금지. 빌드 명령어, 검증 절차, 보안 규칙이 여기에 있음.

### 3단계: 현황 파악 및 다음 작업 안내
- 완료된 작업 목록 확인
- 진행 중인 작업 확인
- **다음 개발할 기능을 구체적으로 안내**
- 관련 설계 문서 경로 제시
- 참조해야 할 기존 코드 경로 제시

## 절대 금지 사항

**다음과 같은 응답은 절대 금지:**
- "어떤 작업을 도와드릴까요?"
- "무엇을 하시겠습니까?"
- 사용자에게 작업 선택을 요청하는 모든 질문

**Claude가 스스로 판단하여 다음 작업을 안내해야 합니다.**

## 핵심 규칙 요약

### 절대 금지
- **운영서버 직접 수정 금지** → 배포 스크립트만 사용
- **샘플/가짜 데이터 사용 금지** → 모든 데이터는 DB에서 API로
- **alert(), toast.success() 금지**
- **POS 관련 파일/DB/PM2 절대 건드리지 않기**

### 개발 워크플로우
```bash
# 1. 개발서버에서 작업
/opt/planq/dev-backend/
/opt/planq/dev-frontend/

# 2. DB 스키마 변경 시
cd /opt/planq/dev-backend && node sync-database.js
pm2 restart planq-dev-backend

# 3. 프론트엔드 빌드 (반드시 run_in_background: true)
cd /opt/planq/dev-frontend && npm run build

# 4. 백엔드 변경 시
pm2 restart planq-dev-backend
```

### API 응답 형식
```javascript
// 성공
res.json({ success: true, data: result });

// 실패
res.status(400).json({ success: false, message: 'Error message' });
```
