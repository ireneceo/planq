# 임시저장 — Clear 전 빠른 저장

**목적:** `/clear` 직전에 **30초 이내**로 현재 상태를 저장하고 바로 이어서 작업할 수 있게 준비.

`/저장` 과 다른 점: 11단계 체크리스트 건너뛰고 **핵심 3가지만** 처리.

---

## Claude 필수 수행 (단순 · 신속)

### 1. Git 자동 커밋 (변경사항 있으면)

```bash
cd /opt/planq && git status --short
```

**변경사항 있으면:**
```bash
cd /opt/planq && git add -A && git commit -m "$(cat <<'EOF'
chore: 세션 중간 저장 - [한 줄 작업 요약]

Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>
EOF
)" && GIT_SSH_COMMAND="ssh -i ~/.ssh/id_ed25519_planq -o IdentitiesOnly=yes" git push origin main
```

- 커밋 메시지는 **한 줄 요약만** (복잡한 본문 X)
- 변경사항이 없으면 이 단계 건너뜀

**변경사항 없으면:** "커밋 스킵 — 변경 없음"

---

### 2. session-state.md 핵심 업데이트 (Write 도구로)

`/opt/planq/.claude/session-state.md` 에 **최소 필수 정보만** 덮어쓰기:

```markdown
## 현재 작업 상태
**마지막 업데이트:** [YYYY-MM-DD HH:MM]
**작업 상태:** 중단 (이어서 재개 예정)

## 📂 다음 할 일 (우선순위 유지)

기존 session-state.md 의 "다음 할 일" 섹션이 여전히 유효하면 그대로 유지.
변경된 게 있으면 해당 항목만 수정.

---

## 🔑 환경변수 / 인증 현황

[변경사항 있으면 기록, 없으면 유지]
```

**기존 섹션 중 다음은 유지**:
- "환경변수 / 인증 현황"
- "다음 할 일 (우선순위 순)"
- "주요 문서 위치"
- "복구 가이드"

**새로 작성하는 부분은 이것만**:
- "현재 작업 상태" 시간 갱신
- "지금 중단 지점" 업데이트
- "이번 세션 작업 요약" 갱신

## 절대 하지 말 것

- ❌ 헬스체크 돌리지 말 것 (오래 걸림)
- ❌ 빌드 돌리지 말 것
- ❌ pm2 restart 하지 말 것
- ❌ 11단계 체크리스트 전부 돌리지 말 것 (그건 `/저장` 의 역할)
- ❌ DEVELOPMENT_PLAN.md 히스토리 섹션 추가하지 말 것 (그건 `/개발완료` 의 역할)
- ❌ Memory 파일 업데이트하지 말 것 (그건 `/개발완료` 의 역할)
- ❌ 장황한 보고하지 말 것
- ❌ 사용자 피드백 기다리지 말 것 — 3단계 끝나면 바로 "완료" 알리기

---

## 목표 소요 시간

**30초 이내.** 넘어가면 `/저장` 쓰는 게 맞음.

## 사용 시나리오

- ☕ 점심/저녁 먹고 옴
- 🚶 자리 비우기 전
- 💤 다음 날 이어서 할 때
- 🧹 대화 컨텍스트 너무 길어서 clear 하고 싶을 때
- 🪟 창 닫기 전 빠른 마무리

## `/저장` vs `/임시저장` vs `/개발완료` 비교

| 명령어 | 소요 | 커밋 | state 상세 | DEV_PLAN 히스토리 | Memory | 헬스체크 | 백업 |
|---|---|:-:|---|:-:|:-:|:-:|:-:|
| `/임시저장` | 30초 | 자동 | 핵심만 | ❌ | ❌ | ❌ | ❌ |
| `/저장` | 2~3분 | 자동 | 전체 | ❌ | ❌ | ❌ | ❌ |
| `/개발완료` | 5~10분 | 자동 | 전체 | ✅ | ✅ | ✅ | ✅ |

**일상 workflow:**
- 잠깐 자리 비울 때 → `/임시저장`
- 세션 끝날 때 (가벼운) → `/저장`
- 하루 끝 · 마일스톤 · 배포 직전 → `/개발완료`
