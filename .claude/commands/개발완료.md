# 개발 완료 명령어

이 명령어는 개발 세션을 마무리할 때 사용합니다.

---

## 필수 수행 (건너뛰기 금지)

**Claude는 반드시 아래 단계를 순서대로 실행해야 합니다. 어떤 단계도 생략하면 안 됩니다.**

---

### 1단계: 오늘 작업 내용 수집 (Bash 실행 필수)

```bash
cd /opt/planq && git diff --name-only HEAD~5
```

이 명령어로 최근 변경된 파일 목록을 확인하고, 현재 세션에서 수행한 작업을 정리합니다.

---

### 2단계: DEVELOPMENT_PLAN.md 히스토리 추가 (Edit 도구 필수 사용)

**반드시 Edit 도구로 `/opt/planq/DEVELOPMENT_PLAN.md` 파일을 수정해야 합니다.**

#### 2-1. 최종 업데이트 날짜 변경

파일 상단의 날짜를 오늘 날짜로 변경:
```
> **최종 업데이트:** 2026-XX-XX
```

#### 2-2. 완료된 작업 섹션 추가

```markdown
## ✅ 완료: [작업 제목] (YYYY-MM-DD)

### 완료된 작업

| 작업 | 설명 | 상태 |
|------|------|:----:|
| [작업1] | [설명] | ✅ 완료 |
| [작업2] | [설명] | ✅ 완료 |

### 수정된 파일
- `path/to/file1.tsx`
- `path/to/file2.js`

---
```

---

### 3단계: 관련 문서 검토 및 업데이트 (Read + Edit 도구 필수 사용)

**오늘 작업과 관련된 문서가 있으면 반드시 업데이트해야 합니다.**

#### 3-1. 변경 파일 기반 관련 문서 자동 매칭

| 변경 영역 | 관련 문서 |
|-----------|----------|
| UI 컴포넌트/스타일 변경 | `/opt/planq/dev-frontend/UI_DESIGN_GUIDE.md` |
| 색상 변경 | `/opt/planq/dev-frontend/COLOR_GUIDE.md` |
| DB 스키마 변경 | DEVELOPMENT_PLAN.md에 기록 |
| 역할/권한 변경 | CLAUDE.md 역할 체계 확인 |
| CLAUDE.md/워크플로우 | `/opt/planq/CLAUDE.md` |

#### 3-2. 검토 및 업데이트 절차

1. 매칭된 문서를 **Read 도구로 읽기**
2. 오늘 변경사항과 문서 내용이 **불일치하는 부분** 찾기
3. 불일치가 있으면 **Edit 도구로 문서 업데이트**
4. 불일치가 없으면 "검토 완료, 업데이트 불필요" 기록

#### 3-3. Memory 파일 업데이트

오늘 작업 중 반복 패턴이나 중요한 아키텍처 결정이 있었으면:
- `/home/irene/.claude/projects/-opt-planq/memory/MEMORY.md` 업데이트 검토
- 기존 내용과 중복되지 않게, 새로운 정보만 추가

---

### 4단계: session-state.md 업데이트 (Write 도구 필수 사용)

**반드시 Write 도구로 `/opt/planq/.claude/session-state.md` 파일을 덮어써야 합니다.**
**"다음 할 일"에는 DEVELOPMENT_PLAN.md의 다음 미완료 항목을 구체적으로 기재합니다.**

- [다음 예정 작업]

### 5단계: 업데이트 검증 (Read 도구 필수 사용)

**반드시 Read 도구로 업데이트된 내용을 확인해야 합니다:**

1. `/opt/planq/DEVELOPMENT_PLAN.md` 상단 50줄 읽기 → 오늘 날짜와 새 섹션 확인
2. `/opt/planq/.claude/session-state.md` 읽기 → 오늘 작업 목록 확인
3. 3단계에서 업데이트한 문서가 있으면 해당 부분 읽기 → 변경 반영 확인

**검증 실패 시 2~4단계 다시 수행**

### 6단계: Git 커밋 및 푸시 (Bash 실행 필수)

```bash
cd /opt/planq && git add -A && git status
```

변경사항 확인 후 커밋:

```bash
git commit -m "$(cat <<'EOF'
chore: Auto-commit YYYY-MM-DD HH:MM:SS

[오늘 작업 요약]
- 작업1
- 작업2

Co-Authored-By: Claude Opus 4.6 (1M context) <noreply@anthropic.com>
EOF
)"
```

푸시:
```bash
GIT_SSH_COMMAND="ssh -i ~/.ssh/id_ed25519_planq -o IdentitiesOnly=yes" git push origin main
```

### 7단계: 백업 생성 (Bash 실행 필수)

```bash
BACKUP_DIR="/opt/planq/backups/dev-daily/$(date +%Y%m%d)"
mkdir -p "$BACKUP_DIR"

tar -czf "$BACKUP_DIR/dev-backend.tar.gz" \
  --exclude='node_modules' \
  -C /opt/planq dev-backend

tar -czf "$BACKUP_DIR/dev-frontend.tar.gz" \
  --exclude='node_modules' \
  --exclude='dist' \
  -C /opt/planq dev-frontend

# 7일 이상 된 백업 정리
find /opt/planq/backups/dev-daily -type d -mtime +7 -exec rm -rf {} + 2>/dev/null

echo "백업 완료: $BACKUP_DIR"
```

## 주의사항

1. **"분석 중...", "업데이트 중..." 같은 말만 하고 실제 도구를 사용하지 않으면 안 됨**
2. **Edit/Write 도구 사용 없이 "업데이트했습니다"라고 하면 거짓말임**
3. **검증 단계(5단계)에서 업데이트가 안 됐으면 다시 해야 함**
4. **운영서버 배포는 별도 `/배포` 명령어 사용**
