## 배포 완료 후 버전 업데이트 + 릴리즈 노트 출력

### 버전 업데이트 (Irene에게 확인)

배포가 성공하면 **Irene에게 버전을 올릴지 물어본다.**

- **올린다고 하면**: patch 버전 +1 → session-state.md, DEVELOPMENT_PLAN.md 반영 + 릴리즈 노트 출력
- **안 올린다고 하면**: 버전 유지, 릴리즈 노트 생략

### 릴리즈 노트 출력

#### 작성 규칙
1. 이번 배포에 포함된 주요 변경사항을 DEVELOPMENT_PLAN.md와 git diff에서 수집
2. 한글 + 영문 둘 다 작성
3. 이모지 넘버(1️⃣ 2️⃣ 3️⃣ 4️⃣)와 줄바꿈으로 구조화
4. 구분선은 `━━━━━━━━━━━━━━━━━━` 사용

#### 출력 형식

```
📢 PlanQ Update v{버전}
YYYY년 M월 D일

━━━━━━━━━━━━━━━━━━

1️⃣ [기능명]

[설명 한 줄]

2️⃣ [기능명]

[설명 한 줄]

━━━━━━━━━━━━━━━━━━

📢 PlanQ Update v{버전}
Month DD, YYYY

━━━━━━━━━━━━━━━━━━

1️⃣ [Feature Name]

[Description line]

━━━━━━━━━━━━━━━━━━
```
