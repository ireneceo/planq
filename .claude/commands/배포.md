## 2단계 — 실제 배포

```bash
cd /opt/planq && ./scripts/deploy-planq.sh
```

스크립트 진행 단계 (~3-5분):

1. preflight (dev 헬스 + SSH 도달)
2. show_changes (변경 파일 목록 + 사용자 확인 1회)
3. backup (운영 backend/frontend-build/q-note → /opt/planq/backups/{TIMESTAMP})
4. sync_backend (rsync dev-backend → 운영, .env 보호)
5. sync_frontend (npm run build + rsync → 운영 frontend-build)
6. sync_qnote (rsync q-note → 운영, --skip-qnote 옵션 가능)
7. install_deps (운영서버 npm ci --production)
8. sync_database (운영서버 node sync-database.js)
9. restart_server (PM2 reload planq-prod-backend / planq-prod-qnote)
10. reload_nginx (운영서버 sudo systemctl reload nginx — sudo 없으면 skip + Irene 안내)
11. verify (https://planq.kr/api/health 200 + 27/27 헬스체크 항목)
12. update_record (이번 배포 timestamp + commit hash 기록)

**플래그**:
- `--auto` — 확인 프롬프트 생략 (검증된 재배포 / 핫픽스)
- `--dry-run` — 미리보기만
- `--skip-build` — 백엔드만 (긴급 hotfix)
- `--skip-qnote` — Q Note Python 서비스 제외

## 4단계 — 배포 후 옵션 (Irene 에게 확인)

### 버전 업

```
"버전 올릴까요? (현재: x.y.z → 다음 patch x.y.{z+1})"
```

- 올린다고 하면:
  - `dev-backend/package.json` version 패치 +1
  - `dev-frontend/package.json` version 동일하게 맞춤 (현재 0.0.0 → 1.0.x 로 통일)
  - `DEVELOPMENT_PLAN.md` 상단 "최종 업데이트" 줄에 v버전 기록
  - `.claude/session-state.md` 마지막 업데이트 + 배포 commit/timestamp 기록
  - 한/영 릴리즈노트 출력 (아래 형식)
- 안 올린다고 하면: 버전 유지, 릴리즈노트 생략

### 릴리즈노트 출력 형식

이번 배포에 포함된 변경사항을 `git log {이전 prod 배포 commit}..HEAD --oneline` 또는 `DEVELOPMENT_PLAN.md` 최근 사이클에서 수집해 한/영 둘 다 작성.

```
📢 PlanQ Update v{버전}
YYYY년 M월 D일

━━━━━━━━━━━━━━━━━━

1️⃣ [기능명]

[설명 한 줄]

2️⃣ [기능명]

[설명 한 줄]

━━━━━━━━━━━━━━━━━━

📢 PlanQ Update v{버전}
Month DD, YYYY

━━━━━━━━━━━━━━━━━━

1️⃣ [Feature Name]

[Description line]

━━━━━━━━━━━━━━━━━━
```

규칙:
- 한글 + 영문 둘 다 필수
- 이모지 넘버(1️⃣ 2️⃣ 3️⃣ 4️⃣)와 줄바꿈으로 구조화
- 구분선은 `━━━━━━━━━━━━━━━━━━`
- 사용자에게 보일 항목만 (내부 리팩토링/디펜던시 업 제외)

## 실패 시

배포 어떤 단계에서든 에러 → 자동으로 다음 단계 차단 (`set -euo pipefail`). 즉시 Irene 에게 출력 그대로 보고하고:

- restart_server / reload_nginx 단계 실패 → 운영 서비스가 reload 전 상태로 살아있음. 안전.
- sync 단계 실패 → 운영 코드는 이전 상태. /opt/planq/backups/{TIMESTAMP} 에 백업 보존.
- verify 실패 → PM2 가 새 코드로 떴지만 헬스체크 실패. `pm2 logs planq-prod-backend --lines 50` 으로 원인 파악 후 결정 (롤백할지 hotfix 할지).

롤백 명령 (필요 시 Irene 승인 받고):
```bash
ssh irene@87.106.78.146 "cd /opt/planq/backups/{TIMESTAMP} && rsync -a backend/ /opt/planq/backend/ && pm2 reload planq-prod-backend"
```
